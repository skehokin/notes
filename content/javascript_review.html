Web Dev Notes - Javascript Review

Summary:

Javascript is the front-end language of the web, central to coding in a browser. Via Node.js, it has also become a viable server-side scripting language, indicating a growing reliance on Javascript for all forms of web development.

Here I teach-to-learn a variety of new and old Javascript ideas.

document.querySelector

When I first started learning Javascript, I was initially taught to access an HTML element like this: 

document.getElementById("element"); // "element" being the ID assigned to the HTML element

Later, I learned that it was in fact possible to refer to elements by their class rather than ID: 

document.getElementsByClassName("element");
which was good because I had heard that IDs were undesirable somehow.

I found this extended syntax a wonderfully persuasive argument for switching to always using jQuery and its rather simpler version: 

$("#element")
and 
$(".element")

As people began to focus on program speed, jQuery became a less valuable option, however. Happily, there is another, slightly less "clunky", version available in plain Javascript: 

 document.querySelector("p.class")

Now, ever since I started learning HTML at 15, it has seemed that web standards have advanced and changed for almost arbitrary reasons. It was easy for me to assume that the new querySelector was the equivalent of using <strong> instead of <b> - the result of a change in ideological approach, and functionally the same.

However, both getElementbyID and querySelector are "up to date" Javascript and do in fact vary more from one another than one might expect.
